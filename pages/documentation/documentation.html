This page displays the OData specifications for OData version 4.0, version 3.0 and version 2.0. The latest version of OData, version 4.0 (plus errata 02), has been publicly standardized by OASIS and is beginning the standardization process with ISO.So we highly recommend all new OData implementations use this version of OData V4. You can take a look at What's new in OData Version 4.0 (PDF version).

To get started learning OData (especially for OData V4), we recommend first working through the basic tutorial and then, read protocol part for an overview of OData. For other parts of the specification:

  • URL conventions describe details on URL syntax
  • Common Schema Definition Language describe details on OData modeling language
  • JSON format and Atom format describe details on payload formats.
  • ABNF defines the URL syntax and serialization format for primitive literals in payloads.
  • Vocabularies describes how to annotate metadata and instance data. (vocabulary page)
  • Aggregation Extension is the extentsion part of OData protocol.
  • Batch processing and operations are sperate documented in OData V2 but covered in Protocol part in OData V3 and V4.
